# 技能审查检查清单

设计或审查 Agent 技能时逐项对照。

## 架构
- [ ] 职责是否单一？删掉别的技能后能否独立工作？
- [ ] 是否碰了下层的实现细节？
- [ ] 是否传了会过时的内容（应该传指针不传内容）？
- [ ] 入口文件是否 ≤ 200 行？超过 500 行是否拆分为 references/？

## 交互
- [ ] 所有确认点是否使用 AskUserQuestion 选择框？
- [ ] 推荐项是否标注了 `(推荐)` + 理由？
- [ ] 其他选项是否标注了不推荐的理由？
- [ ] 是否禁止了自由文本输入和开放式追问？

## 质量
- [ ] 每个验收条件是否绑定了具体门禁维度？
- [ ] 失败处理是否按 A/B/C/🎨 根因分类？
- [ ] 是否有退出机制（stagnated/exhausted）？
- [ ] 是否有终态保护（不会无限循环）？

## 运行
- [ ] Token 优化工具是否在执行层（不是在编排层）？
- [ ] 降级链路是否完整（首选→备选→兜底）？
- [ ] 决策是否写入 decision_log？
- [ ] 关键状态是否持久化（状态文件 + heartbeat 并发检测）？

## 文件结构
- [ ] 共享组件是否提取为独立文件？
- [ ] 模式分发是否清晰（主文件路由 + references/ 详情）？
