# A Philosophy of Software Design 规则集

当进行软件设计、模块划分或需要降低系统复杂度时使用此技能。

## 使用方式

加载此技能后，**必须**读取并严格遵循以下规则文件：

```
C:\Users\admin\.agents\skills\agent-rules-books\a-philosophy-of-software-design\a-philosophy-of-software-design.mini.md
```

## 规则概览

- 复杂度是增量累积的
- 深模块 vs 浅模块：接口简洁、实现复杂
- 信息隐藏：模块封装非显而易见的设计决策
- 通用化与专用化的平衡
- 分层设计
- 注释应描述不明显的事情
- 异常处理的策略
- 设计即持续投资

**_规则文件是此技能的核心。务必在设计前读取它。_**
