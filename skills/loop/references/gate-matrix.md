# 门禁矩阵（G1-G8, F1-F5, G9-G10）

门禁矩阵是 loop 的质量底线——每轮编码后强制执行，所有 ❌ 必须进自愈闭环。

## 维度启用规则

| 场景 | 启用维度 |
|------|---------|
| 纯后端 API | G1-G8, G9（G1 必选，纯 Python 时 G3 独立启用） |
| 含前端页面 | **G0** + G1-G8, G9 + F1-F4 |
| 含前端+样本URL | **G0** + G1-G8, G9 + F1-F5 |
| 纯配置/文档 | G2, G7, G9（轻量） |

> **G9/G10 与 go 的协作**：
> - **G9（代码审查）** 在 loop 每个子任务后自动运行——**每个子任务都审查**
> - **G10（系统审查）** 在 go 特性分支合并前运行——**每个特性分支审查一次**，不在 loop 中重复触发
> - loop 不调用 system-review 技能，仅调用 code-reviewer 技能

## 维度定义

### G0 agent-browser 环境 🔴 阻塞红线（前端任务强制前置）
- **检查**: `bash skills/loop/scripts/check-agent-browser.sh` 退出码 = 0
- **判据**: CLI 安装 + 版本 ≥ 0.29.0 + Chrome 可用 + 启动成功 + MCP 配置正确
- **失败处置**: 阻塞汇报，提示修复命令，禁止继续前端验证
- **红线**: 环境不通 ≠ 验证通过，必须先修通
- **跳过条件**: 纯后端/纯文档任务（不涉及 F1-F5）

### G1 依赖可达 🔴 阻塞红线
- **检查**: DB 可达、Redis 可达、外部 API 可达
- **判据**: 实测连通，禁止"假设能连"
- **失败处置**: 三级回退: docker-compose → Docker daemon → 阻塞汇报
- **红线**: 环境不通≠验证通过，必须先修通

### G2 编译 🟡 全自动修
- **检查**: tsc / mypy 0 错误
- **判据**: 0 error，编译成功
- **失败处置**: 自动 systematic-debugging 修复

### G3 类型 🟡 全自动修
- **检查**: 类型检查（可并入 G2 或独立）
- **判据**: 0 error
- **失败处置**: 自动修复

### G4 单测 🔴 全自动修
- **检查**: pytest / jest 全量通过
- **判据**: 全部通过 + 覆盖率≥阈值（默认80%）
- **失败处置**: 自动 systematic-debugging

### G5 集成冒烟 🔴 全自动修
- **检查**: 后端起服 + HTTP 请求
- **判据**: 全部 2xx
- **失败处置**: A 类 debug+修 / C 类重启服务
- **红线**: 必须对真实后端发请求，禁止 mock 替代

### G6 契约 🟡 全自动修
- **检查**: 前后端字段一致
- **判据**: 0 差异
- **失败处置**: 自动修复

### G7 构建 🟡 全自动修
- **检查**: vite build / 打包
- **判据**: 构建成功

### G8 Docker 🟡 全自动修
- **检查**: 镜像构建 + 启动
- **判据**: 容器健康检查通过

### F1 控制台 🟡 分级触发
- **检查**: `agent_browser_errors` + `agent_browser_console`
- **判据**: error 数量 = 0（warning 不计入 F1 失败）
- **红线**: console error 数组长度=0，禁止肉眼截图判断

### F2 网络 🟡 分级触发
- **检查**: `agent_browser_network_requests`
- **判据**: 全部 2xx/3xx
- **红线**: 逐条状态码检查，4xx/5xx 逐条归类

### F3 渲染 🟡 分级触发
- **检查**: `agent_browser_snapshot` 元素存在（@eN 紧凑引用）
- **判据**: 验收条件要求的元素全部命中

### F4 交互 🟡 分级触发
- **检查**: 操作流每步断言
- **判据**: 全部通过
- **红线**: 每步 snapshot 断言，禁止"点了没崩就算过"

### F5 样本对照 🟡/🟢 分级
- **检查**: 实际产物 vs 样本基准结构
- **判据**: 
  - 🟡 客观项（区块缺失/元素类型错）→ 自动判定，进自愈
  - 🟢 主观项（配色/间距/风格）→ 标记"🎨 设计待确认"，不自动改

### G9 代码审查 🔴 全自动修（每子任务触发）
- **检查**: 加载 `code-reviewer` 技能，对本次 commit 改动的文件做代码层审查
- **检查范围**: `git diff HEAD~1` 的所有变更文件
- **判据**: 无 ERROR 级别问题
- **失败处置**: 
  - ERROR → 任务标记为 failed，触发 self-healing 重试
  - WARNING → 记录到 handoff 摘要，不阻断流程
- **职责区分**:
  - G9 (loop 内) = 单子任务的**代码层**审查（安全漏洞、代码异味）
  - G10 (go 内) = 整个特性分支的**系统层**审查（接口矛盾、分层合理性）
  - **不重复调用**：loop 只跑 G9，go 只跑 G10，文件集合天然不重叠
- **跳过条件**: L1 轻量模式可跳过 G9（用户决定）

### G10 系统审查 🔴 一次性（特性分支级，不在 loop 重复）
- **说明**: G10 实际在 go 的 **Step ⑦.5** 交付前触发，**不在 loop 的门禁矩阵中执行**
- **loop 中只需声明**: G10 的检查结果会通过 `handoff.gate_result` 字段回传，loop 在 Step 6 读取并展示
- **职责**: 见 `skills/system-review/SKILL.md`，调用的是 system-review 技能，不是 G10 本地函数

## 环境前置探测（G1 维度）

G1 是所有非纯文档任务的**强制前置**:

1. **DB 可达性**: 实测连接
2. **Redis 可达性**: `redis-cli ping` 或等价方式
3. **外部 API 可达性**: `curl` 关键端点

不通时按三级回退:
1. docker-compose up -d <db> 起本地依赖
2. 检查 Docker daemon → 尝试启动
3. 失败 → 阻塞保护，告知用户，禁止继续声称验证通过
