# 提示词模板：动态来源分级

> 用于 deep-research（深度调研） 的 **Search 阶段**。对每个来源做 T1/T2/T3/Reject 动态分级。

---

## 用途

让 AI 在 Search 阶段自动评估每个来源的可信度，避免被"看起来权威但实际是营销内容"的来源污染。

## 完整提示词

```
你是 deep-research（深度调研） 的 Search 阶段来源分级员。请对每个来源做动态分级。

分级标准（按此顺序判断）：

1. **T1（一手 / 高权重）**
   - 同行评审论文、学术书籍
   - 官方文档（库 / 框架 / API 官方）
   - 官方标准（W3C / RFC / ISO / WHO）
   - 一手档案（政府公开数据、原始数据集）
   - 源码（直接读 .py / .js / .md 原文）

2. **T2（权威二手 / 中权重）**
   - 官方仓库的 README / CHANGELOG / Issues
   - 可识别作者的博客（作者在该领域有公认工作）
   - 录制会议（有讲者介绍 + 主题明确）
   - 有引用的 Wikipedia 条目
   - 有方法论的行业报告

3. **T3（低质 / 低权重）**
   - 引用 T1/T2 的博客
   - 高赞论坛答案（Stack Overflow 等）
   - 可识别专家的访谈
   - 个人技术博客（无引用链）

4. **Reject（拒收）**
   - 无作者 / 无数据 / 无引用
   - 营销内容（产品推销 / SEO 软文）
   - 未监督 AI 生成内容（无人工审核痕迹）
   - 未注明来源的教程
   - 内容农场 / 列表型 "Top 10" 文章

对每个来源，输出：
- [T1/T2/T3/Reject] 标签
- 一句话分级理由
- 该来源对该子问题的支持力度（强/中/弱）
```

## 使用时机

Search 阶段收集完每个来源的 URL + 摘要后，**逐个**用此提示词分类。

## 示例输入

```
来源：https://arxiv.org/abs/2301.12345
标题：Deep Learning for Code Review
摘要：Peer-reviewed paper proposing novel architecture...
```

## 示例输出

```
[T1] Peer-reviewed paper on arXiv（一手学术发表）
支持力度：强
```

## 与 recursive-research 的差异

| 维度 | recursive-research | 本模板 |
|------|------------------|--------|
| 分级策略 | 4 级严格（T1/T2/T3/Reject）| 4 级动态（AI 决定）|
| 来源元数据 | 来源 URL + 作者 + 发布日期 | 来源 URL + 摘要 + AI 判断 |
| 质量保证 | 来源必须可追溯 | AI 判断可错（**已知限制**）|

## 已知限制

- ⚠️ AI 分级可能错（特别是 T2 vs T3 边界）
- ⚠️ "Reject" 标签需要勇气——AI 倾向"宽容"
- ⚠️ 动态分级不可复现（不同次跑结果可能不同）

**缓解**：在 WDM 阶段显式标注"已知限制"；Report 引用清单按 T1/T2/T3 分组展示。
