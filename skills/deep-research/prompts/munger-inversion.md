# 提示词模板：Munger 反向自检

> 用于 deep-research（深度调研） 的 **Reason Munger 阶段**。对 WDM 胜出方案做 5 问反向自检。

---

## 用途

基于 Charlie Munger 引用 Jacobi 的"反过来想"哲学。对 WDM 胜出的方案做反向自检，避免"成功时高估、失败时低估"的认知偏差。

## 完整提示词

```
你是 deep-research（深度调研） 的 Reason Munger 阶段反向自检员。请对 WDM 胜出的方案做 5 问反向自检。

**WDM 胜出方案**：<从 20-reason-wdm.md 复制>

5 问反向自检（每问必答，每答不少于 2 句话）：

1. **它会如何失败？**
   列出 3-5 个最可能的失败模式。严重度分 🟢/🟡/🟠/🔴。

2. **它有什么偏见？**
   来源支持了它的哪些方面？忽略了哪些方面？是否有"自我实现的预言"？

3. **我忽略了什么？**
   WDM 没考虑但实际重要的因素。提示：边界条件、长期影响、二阶效应。

4. **谁会反对这个结论？为什么？**
   列出 2-3 个可能的反对者。理由要具体，不要"某某不认同"。

5. **1 年后回头看，这个结论会让我尴尬吗？**
   诚实优先于正确。列出 1-3 个"可能尴尬"的点。

输出格式：
- 每问独立段落
- 总长 200-500 字
- 末尾追加"自我评估"：胜出方案强度 + 真正风险 + 行动调整

**关键铁律**：
- 诚实优先于正确
- 不要"粉饰"——列出真正的风险
- 失败模式要具体（不是"可能失败"）
```

## 使用时机

WDM 阶段产出 20-reason-wdm.md 后立即使用。

## 示例输入

```
WDM 胜出方案：方案 B（按 3 层架构组织）— 4.60 分
```

## 示例输出

```
1. 它会如何失败？
   - 失败模式 1：用户不熟悉 v5.4 演进，看不懂"3 层"含义
     严重度：🟡 中（需 1-2 段铺垫）
     缓解：演进时间线前置
   ...

5. 1 年后回头看？
   - v7.0 推出后"3 层架构"作废
     概率：🟡 中
     缓解：报告加"截至 2026-06-29 版本"声明

自我评估：
- 胜出方案强度：总分 4.60，领先方案 A 0.60
- 真正风险：alpha mock 当真实能力介绍
- 行动调整：Report 必须显式标注"alpha 阶段"
```

## Munger 哲学要点

- **"反过来想，总是反过来想"** —— Charlie Munger
- **承认自己不知道什么，比宣称知道更重要**
- **决策的反向思考比正向思考更有价值**

## 已知限制

- ⚠️ "自我评估"环节容易出现"AI 自我感觉良好"
- ⚠️ 5 问可能流于形式（特别是失败模式 1 容易敷衍）
- ⚠️ Munger 本人强调"lollapalooza effect"（多因素叠加），本模板未覆盖

**缓解**：在严重度评估时要求**至少 1 个 🟠 或 🔴**；否则认为反向自检不够深。
