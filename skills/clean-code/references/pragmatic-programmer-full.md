# The Pragmatic Programmer 规则集

当需要培养务实的工程习惯、作出技术决策或提升开发效率时使用此技能。

## 规则概览

- DRY：不要重复自己
- 正交性：模块间保持独立
- 可逆性：没有最终决定
- 曳光弹：逐步逼近目标
- 原型：用原型探索风险
- 领域语言：用 DSL 表达问题
- 估算：学会估算并记录
- 注重实效：足够好即可
- 知识组合：持续学习投资
- 交流：与团队和用户高效沟通
