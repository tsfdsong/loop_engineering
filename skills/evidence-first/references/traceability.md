# 追溯链规范

> 本文档定义 evidence-first 技能对"结论可追溯"的要求。
> **核心原则**：每条结论必须可追溯到具体事实。

---

## 追溯链模型

```
[结论] ← 必须可追溯到 → [事实 F] + [来源]
   ↑                        ↑      ↑
   │                        │      └─ 文件路径 / git hash / 命令输出
   │                        └─ 已验证的事实
   └─ 回答给用户的最终结论
```

---

## 追溯链要求

### 必填项

每条结论必须包含：

| 字段 | 说明 | 示例 |
|------|------|------|
| **结论** | 一句话清晰陈述 | "v5.4 不应现在删" |
| **事实依据** | 至少 1 个 [F] | "[F] v54-baseline.json 27 条 trace 用于回归保护" |
| **来源** | 具体的文件/命令 | "`cat tests/golden-traces/v54-baseline.json`" |
| **验证方式** | 其他如何复现 | "python -c 'import json; print(len(json.load(open(\"tests/golden-traces/v54-baseline.json\"))))'" |

---

## 追溯链格式

### 强格式（推荐用于关键决策）

```markdown
## 结论

**v5.4 不应现在删**。

## 事实依据

1. **[F]** v5.4 SKILL.md 备份存在
   - 来源: `ls skills/skill-hub/SKILL.md.v5.4.backup`
   - 验证: 文件存在
   
2. **[F]** v54-baseline.json 含 27 条黄金轨迹
   - 来源: `cat tests/golden-traces/v54-baseline.json | python -c "import json,sys; print(len(json.load(sys.stdin)))"`
   - 验证: 输出 27
   
3. **[F]** 当前 main 分支最新提交提到 v5.4
   - 来源: `git log --oneline | grep v5.4`
   - 验证: 找到匹配行

## 反事实

如果 v5.4 被删除:
- 27 条回归测试失效 ← 依据 [F]2
- v6.0 改动无回归保护 ← 依据 [F]2
- ZCode 加载路径可能再次出 bug（基于 cedb61c 提交）← [F] git log
```

### 弱格式（用于一般论述）

```markdown
v5.4 不应删，因为 27 条 baseline 是 v6.0 回归保护（[F] v54-baseline.json）。
```

---

## 反事实分析（推荐）

每个关键结论应做**反事实分析**：

| 步骤 | 说明 |
|------|------|
| 1. 提出结论 | "v5.4 不应删" |
| 2. 列事实依据 | [F] 1, 2, 3 |
| 3. 反事实 | "如果删了会怎样？" |
| 4. 验证反事实 | 每个反事实是否可追溯 |

### 示例

```
结论: v5.4 不应删
反事实: 如果删了
  - baseline 文件消失 → 27 条回归测试失效 → 失去保护
    ↑ 依据: [F] baseline 文件含 27 条
  - ZCode 加载路径参考消失 → 可能再次出 bug
    ↑ 依据: [F] git log cedb61c 提到 v5.4 bug
  - v6.0 设计文档的对比基线消失 → 设计追溯困难
    ↑ 依据: [F] docs/2026-06-29-skill-hub-v6-design.md
```

---

## 追溯链强度分级

| 强度 | 要求 | 适用 |
|------|------|------|
| **强** | 多个 [F] + 反事实分析 | 关键决策（删/改/合并） |
| **中** | 至少 1 个 [F] | 重要建议 |
| **弱** | 多个 [H]/[P] | 一般讨论 |

**关键决策必须强追溯**。

---

## 追溯链可视化

```
v6.0 不能删 v5.4 (结论)
├─ [F] v5.4 SKILL.md 备份存在 (ls)
├─ [F] v54-baseline.json 27 条 (cat)
├─ [F] cedb61c 提交提到 v5.4 (git log)
├─ [F] docs/v6-design.md 提到 v5.4 基线 (cat)
│
└─ 反事实: 删了会怎样？
   ├─ 失去 27 条回归保护 ← [F] 2
   ├─ ZCode 加载路径 bug 风险 ← [F] 3
   └─ 设计追溯困难 ← [F] 4
```

---

## 追溯链断裂 = 不可信

### 常见断裂

| 断裂 | 修复 |
|------|------|
| 结论 → 多个 [H] | 至少升级 1 个为 [F] |
| 结论 → 多个 [P] | 找到本项目的具体事实 |
| 结论 → 无标注 | 标注 [F]/[H]/[P] |
| 结论 → 套用通用经验 | 改为本项目具体事实 |

---

## 追溯链自检

回答后自问：

- [ ] 每条结论都标注了吗？
- [ ] 每个 [F] 都有可追溯的来源吗？
- [ ] 其他人在相同环境能复现吗？
- [ ] 关键结论做了反事实分析吗？
- [ ] 追溯链完整（结论 ← 事实 ← 来源）吗？

**全部 ✅ → 结论可信**
**任一 ❌ → 重新回答**
